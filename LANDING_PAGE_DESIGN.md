# Final Design — Landing Page Simpel POS

## 1. Ringkasan Pemahaman

- Landing page dibuat untuk **Simpel POS**, berdasarkan repo open source `https://github.com/ryansutrisno/simple-pos-laravel`.
- Target utama adalah **pemilik toko kecil / UMKM**, bukan developer.
- Tujuan utama landing page adalah mendapatkan **traffic organik dari Google** dengan fokus keyword **“POS untuk UMKM”**.
- Alur pengunjung: memahami manfaat dan fitur terlebih dahulu, terutama dashboard, lalu diarahkan ke CTA **Minta Preview via WhatsApp**.
- Identitas teknis seperti Laravel dan open source tidak ditonjolkan di konten utama; GitHub cukup ditampilkan sebagai link kecil di footer.
- Landing page harus sangat cepat, ringan, SEO-friendly, mudah dirawat manual, dan cocok untuk traffic SEO menengah.

## 2. Tujuan

Landing page ini bertujuan membantu pemilik UMKM memahami bagaimana Simpel POS dapat membantu memantau kondisi toko melalui dashboard yang sederhana dan informatif.

Fokus utama bukan langsung menjual, tetapi membangun pemahaman dan kepercayaan sampai pengunjung merasa cukup tertarik untuk meminta preview melalui WhatsApp.

## 3. Target Pengguna

### Pengguna utama

**Pemilik toko kecil / UMKM** yang membutuhkan sistem POS untuk membantu memantau:

- Produk
- Stok
- Transaksi
- Ringkasan keuangan
- Alert reorder
- Performa toko secara cepat

### Bukan target utama halaman awal

- Developer Laravel
- Kontributor open source
- Pengguna yang mencari dokumentasi teknis
- Pengunjung yang langsung ingin clone repo

## 4. Keyword dan SEO

### Keyword utama

**POS untuk UMKM**

### Arah SEO

Landing page harus menjawab intent pengguna yang sedang mencari solusi POS untuk toko kecil atau UMKM, bukan intent teknis seperti “aplikasi POS Laravel” atau “POS open source”.

### Konten pendukung SEO

Gunakan FAQ yang menjawab pertanyaan sederhana seperti:

- Apa itu POS untuk UMKM?
- Apakah Simpel POS cocok untuk toko kecil?
- Apa manfaat dashboard Simpel POS?
- Apakah bisa melihat preview dulu?

## 5. Struktur Halaman

### 5.1 Hero SEO

Hero menjadi titik masuk utama untuk keyword **POS untuk UMKM**.

#### Arah headline

Contoh arah headline:

> POS untuk UMKM yang Bantu Pantau Toko Lebih Mudah

#### Arah subheadline

Simpel POS membantu pemilik toko melihat ringkasan produk, stok, transaksi, alert reorder, dan grafik toko dari satu dashboard yang mudah dipahami.

#### CTA utama

> Minta Preview via WhatsApp

#### CTA sekunder

> Lihat Fitur Dashboard

### 5.2 Preview Dashboard

Gunakan gambar utama:

```text
Dashboard-Simple-Pos-Statistik.png
```

Alasan pemilihan:

- Lebih sederhana dibanding screenshot dashboard lengkap.
- Lebih terang dan mudah dipahami untuk pemilik toko kecil.
- Cocok sebagai first impression karena langsung menampilkan statistik aplikasi.
- Tidak terlalu padat secara visual.

Screenshot `Dashboard-Utama-Simpel-POS.png` dapat digunakan sebagai visual pendukung jika nanti dibutuhkan untuk bagian fitur yang lebih detail.

### 5.3 Masalah Pemilik Toko

Bagian ini menjembatani hero menuju solusi. Gunakan bahasa praktis dan dekat dengan keseharian pemilik toko.

Masalah yang ditonjolkan:

1. Sulit melihat kondisi toko secara cepat.
2. Stok menipis sering terlambat diketahui.
3. Laporan penjualan belum mudah dibaca.
4. Produk terlaris dan performa toko tidak langsung terlihat.

#### Arah copy

> Banyak pemilik toko perlu melihat stok, transaksi, dan kondisi penjualan dari beberapa tempat berbeda. Simpel POS membantu menyatukan ringkasan penting itu ke dalam satu dashboard.

### 5.4 Solusi / Fitur Dashboard

Bagian ini menjelaskan manfaat dashboard sebagai solusi utama, bukan menjelaskan fitur teknis.

#### Kartu fitur utama

1. **Ringkasan Produk & Stok**  
   Menampilkan kategori, produk, stok, varian, paket aktif, dan alert reorder.

2. **Pantauan Transaksi**  
   Membantu melihat aktivitas transaksi melalui grafik yang lebih mudah dibaca.

3. **Ringkasan Keuangan**  
   Memberi gambaran pemasukan dan pengeluaran secara visual.

4. **Alert Reorder**  
   Membantu pemilik toko mengetahui stok yang perlu diperhatikan sebelum benar-benar habis.

5. **Dashboard Mudah Dipahami**  
   Statistik penting disajikan secara ringkas agar pemilik toko cepat membaca kondisi bisnis.

### 5.5 Kenapa Cocok untuk UMKM

Bagian ini memberi rasa aman bahwa Simpel POS bukan sistem yang terlalu rumit untuk toko kecil.

Poin utama:

- Fokus pada kebutuhan toko kecil.
- Dashboard langsung memberi gambaran kondisi toko.
- Membantu keputusan harian seperti cek stok, restock, dan pantau transaksi.
- Menggunakan bahasa sederhana, bukan bahasa teknis.
- Pengunjung bisa meminta preview dulu tanpa tekanan untuk langsung membeli atau memasang.

### 5.6 FAQ SEO

FAQ digunakan untuk edukasi dan SEO, bukan dokumentasi teknis.

#### FAQ yang disarankan

**Apa itu POS untuk UMKM?**  
POS untuk UMKM adalah sistem kasir yang membantu pemilik usaha mencatat transaksi, memantau produk, melihat stok, dan membaca ringkasan toko dari satu tempat.

**Apakah Simpel POS cocok untuk toko kecil?**  
Ya. Simpel POS cocok untuk toko kecil yang ingin mulai merapikan penjualan, stok, dan laporan sederhana tanpa sistem yang terlalu rumit.

**Apa manfaat dashboard Simpel POS?**  
Dashboard Simpel POS membantu pemilik toko melihat jumlah produk, stok, varian, paket aktif, alert reorder, serta grafik transaksi dan keuangan secara cepat.

**Apakah saya bisa melihat preview dulu?**  
Ya. Pengunjung dapat meminta preview melalui WhatsApp untuk melihat apakah Simpel POS sesuai dengan kebutuhan toko.

### 5.7 CTA Akhir

CTA akhir mengulang ajakan utama setelah pengunjung memahami fitur dan manfaat.

#### Arah copy

> Ingin lihat apakah Simpel POS cocok untuk toko Anda?

#### Button

> Minta Preview via WhatsApp

## 6. Non-Goal

Hal yang tidak dikejar pada versi awal landing page:

- Menjelaskan teknis Laravel.
- Menjelaskan detail open source di konten utama.
- Membuat dokumentasi fitur lengkap per halaman.
- Menyediakan live demo publik.
- Membuat pricing atau paket layanan.
- Membuat form lead dan penyimpanan data pribadi.

## 7. Non-Functional Requirements

### Performance

- Landing page harus sangat cepat.
- Fokus pada Core Web Vitals.
- Gambar perlu dioptimasi agresif.
- Hindari JavaScript berlebihan.
- Prioritaskan halaman statis dan ringan.

### Scale

- Target awal: traffic SEO menengah.
- Estimasi: ribuan pengunjung per bulan.
- Tidak perlu arsitektur kompleks untuk campaign besar pada fase awal.

### Security / Privacy

- Kebutuhan analytics dan tracking ditunda.
- Default awal: tidak ada form lead.
- Tidak ada penyimpanan data pribadi di landing page.
- CTA WhatsApp digunakan sebagai jalur komunikasi utama.
- Jika analytics dipakai nanti, pilih pendekatan ringan dan minim data pribadi.

### Reliability / Availability

- Cukup menggunakan standar static hosting / CDN.
- Tidak perlu monitoring, fallback, atau alert kompleks pada fase awal.

### Maintenance / Ownership

- Konten akan dirawat sendiri oleh pemilik proyek.
- Struktur konten harus sederhana dan mudah diedit manual.
- Tidak perlu CMS atau workflow editorial kompleks.

## 8. Decision Log

| Keputusan | Alternatif | Alasan |
| --- | --- | --- |
| Menggunakan pendekatan one-page educational SEO landing | Multi-page feature docs, landing + banyak halaman fitur | Lebih ringan, cepat, mudah dirawat, dan cukup untuk validasi awal SEO |
| Fokus target ke pemilik toko kecil / UMKM | Developer, kontributor OSS, implementor teknis | Tujuan utama adalah lead pengguna bisnis |
| Keyword utama “POS untuk UMKM” | “Aplikasi POS Laravel”, “software kasir open source”, “dashboard POS” | Lebih sesuai dengan intent bisnis dan organic traffic yang diinginkan |
| CTA utama WhatsApp request preview | GitHub, live demo publik, form lead | Lebih mudah, personal, dan cocok untuk edukasi awal |
| Laravel/open source tidak ditonjolkan | Menonjolkan repo dan stack teknis | User ingin fokus ke manfaat UMKM; GitHub cukup di footer |
| Gambar utama `Dashboard-Simple-Pos-Statistik.png` | `Dashboard-Utama-Simpel-POS.png`, dark/light dashboard | Lebih sederhana dan mudah dipahami sebagai preview awal |
| Per-feature docs ditunda | Membuat halaman dokumentasi fitur sejak awal | Menghindari maintenance tinggi sebelum strategi long-tail SEO jelas |
| Default privacy minimal | Form lead, analytics lengkap, tracking iklan | Security/privacy belum diputuskan; default aman adalah tidak menyimpan data pribadi |
| Perbaikan proporsi & layout Hero | Membalik grid menjadi 1.1fr (teks) vs 0.9fr (gambar) & merampingkan font/margin | Agar seluruh elemen hero (CTA, tombol, card kecil) masuk dalam satu viewport desktop tanpa perlu scroll ke bawah (above the fold). |
| Preview bagian "Cocok untuk UMKM" | Menggunakan video MP4 (autoplay, muted, loop) sebagai ganti screenshot statis penuh | Lebih ringkas secara vertikal dan lebih interaktif (animasi) untuk memamerkan produk tanpa memberatkan halaman (preload metadata). |
| Perbaikan Layout "Cocok untuk UMKM" | Membalik grid ke 1.1fr vs 0.9fr dan menambah min-height viewport | Konsistensi dengan bagian Hero agar seluruh konten fit di satu layar (above the fold) tanpa butuh scroll vertikal. |
| Refactoring Komponen & CSS Scoping | Memecah `index.astro` dan `global.css` ke beberapa file komponen mandiri (`src/components/`) | Membuat kode jauh lebih terstruktur, mudah di-maintain, dan meniru best practice Astro dengan style scoped per section. |
